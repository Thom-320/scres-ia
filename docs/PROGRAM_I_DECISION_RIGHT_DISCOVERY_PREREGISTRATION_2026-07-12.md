# Program I — Decision-Right Discovery and Adaptive-Headroom Map

## Binding question

Which thesis-grounded or explicitly declared Op3–Op13 controls materially move
MFSC outcomes, reverse their action ranking across physical states, and remain
convertible using information available before action?

Program I separates four estimands: physical sensitivity, state-contingent
perfect-information headroom, observable rollout conversion, and learned value.
No earlier estimand substitutes for a later one.

## Two lanes

Lane A searches the Garrido-realistic envelope first. Lane B is opened only by
a terminal Lane A verdict and is reported as an engineered benchmark. Lane B
may vary persistence, advance information, shared-resource scarcity, and
risk–mitigation specialization. It may not select a cell using learner results.

## Sequence

1. Audit the decision-right catalog and implement liveness/resource adapters.
2. Run grouped Morris over implemented numeric controls and a D-optimal design
   over categorical doctrines using identical exogenous tapes.
3. Refine at most ten factors using dependent-input-aware Sobol indices and a
   validated surrogate. Sensitivity is descriptive and cannot authorize RL.
4. Construct three to five equal-right action families and branch 240 states
   exactly under short and long horizons.
5. Test regret-weighted depth-3 trees and hysteresis/lookahead policies by
   tape-cross-fitted sequential rollout against full-contract static and
   periodic comparators.
6. Freeze the lowest-amplitude connected passing condition on fresh tapes.
   Only that freeze may authorize an RL preregistration.

## Claim boundary

`mu_star`, Sobol total effects, or a large output range identify influential
inputs. Adaptive headroom additionally requires material action-ranking
reversal, oracle advantage over the best same-contract constant, practical
service value, resource equivalence, horizon stability, and observable rollout
conversion. Classification accuracy is never an endpoint.

Seeds 1110001+ and 1120001+ remain locked. The present implementation exposes
only the subset of catalog factors supported safely by the shared DES. A factor
marked `requires_adapter` is excluded until its liveness, mass, CRN, and resource
ledger tests pass.
